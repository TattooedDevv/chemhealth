## 1. Overview

This section documents the technology stack selected for ChemHealth and the rationale for each choice.

## 2. Frontend
- React
  - Rationale: component-based UI, strong ecosystem, ideal for dashboards and data visualization

## 3. Backend
- Python (FastAPI or Django)
  - Rationale: strong ML/data ecosystem, rapid API development, maintainability

## 4. Data Science / Machine Learning
- NumPy / Pandas
- Scikit-learn
  - Rationale: reliable baseline models and evaluation; fast iteration for an academic project

## 5. Database
- PostgreSQL
  - Rationale: relational structure fits datasets/results/users; widely used, production-ready

## 6. DevOps / Infrastructure
- Docker / Docker Compose
  - Rationale: reproducible dev environment and consistent deployment
- Jenkins (CI/CD)
  - Rationale: pipeline automation; aligns with DevOps goals

## 7. Cloud (Planned)
- Azure (future deployment target)
  - Possible services: Azure App Service or AKS, Azure Database for PostgreSQL, Azure ML

## 8. Storage
- UGreen NAS
  - Rationale: local dataset storage and backups during development
